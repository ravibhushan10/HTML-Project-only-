HTML Media Tags and Deployment

Media Tags

Media tags in HTML allow us to add multimedia content such as images, audio, video, and other interactive content to a webpage.

1. Image Tag (<img>)

Used to embed images.

<img src="images/photo.jpg" alt="Description" width="300" height="200">
  • src: Path to the image file.
  • alt: Alternate text if the image is not displayed.
  • width, height: Dimensions of the image.

Example of Image tag:

2. Audio Tag (<audio>)

Used to embed audio files.

<audio controls>
  <source src="audio/music.mp3" type="audio/mpeg">
  <source src="audio/music.ogg" type="audio/ogg">
  Your browser does not support the audio element.
</audio>
  • controls: Adds play, pause, volume options.
  • autoplay: Starts audio automatically.
  • loop: Plays audio repeatedly.

Example of audio tag:

3. Video Tag (<video>)

Used to embed video files.

<video width="400" height="300" controls>
  <source src="video/movie.mp4" type="video/mp4">
  <source src="video/movie.ogg" type="video/ogg">
  Your browser does not support the video tag.
</video>
  • controls: Shows play, pause, fullscreen buttons.
  • autoplay: Starts video automatically.
  • loop: Plays video repeatedly.
  • muted: Mutes video by default.

Example of Video tag:

4. Iframe Tag (<iframe>)

Used to embed another webpage, map, or video (like YouTube) inside a webpage.

<iframe src="https://www.example.com" width="600" height="400"></iframe>

<!-- YouTube Embed Example -->
<iframe width="560" height="315"
src="https://www.youtube.com/embed/dQw4w9WgXcQ"
title="YouTube video" frameborder="0" allowfullscreen></iframe>

5. Embed Tag (<embed>)

Used for embedding external content such as PDFs or multimedia files.

<embed src="files/sample.pdf" type="application/pdf" width="600" height="500">

Deployment of HTML

Deployment means making your HTML website accessible on the internet. Steps include:

1. Save Your Project

project/
│
├── index.html
├── about.html
├── images/
│   └── photo.jpg
├── audio/
│   └── music.mp3
└── video/
    └── movie.mp4

2. Test Locally

Open index.html in your browser to check everything works before uploading.

3. Choose Deployment Method

  • Localhost: Open HTML file directly in browser.
  • GitHub Pages: Free hosting with GitHub repository.
  • Netlify / Vercel: Free and easy cloud hosting.
  • Web Hosting: Upload files to hosting provider (Hostinger, Bluehost, etc.).

4. Deployment Example with GitHub Pages

  1. Create a GitHub account.
  2. Create a repository and upload HTML files.
  3. Go to Settings > Pages in repository.
  4. Select branch main and folder /root.
  5. Your website will be live at: https://username.github.io/repository-name

5. Deployment Example :

  1. Go to:
    • Netlify
    • Render
    • Vercel
  2. Sign up and drag-drop your project folder.
  3. Your site will be live with a free domain.
