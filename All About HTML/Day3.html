HTML Boilerplate and File Paths

Boilerplate Code

Boilerplate code is the basic structure of an HTML page. It contains the required tags to set up a webpage properly.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Webpage</title>
</head>
<body>
  <h1>Hello, World!</h1>
  <p>This is the basic HTML boilerplate.</p>
</body>
</html>

Explanation

  • <!DOCTYPE html>: Defines document type as HTML5.
  • <html lang="en">: Root element, language set to English.
  • <head>: Contains meta info, title, links.
  • <meta charset="UTF-8">: Supports all character sets.
  • <meta name="viewport">: Makes page responsive.
  • <title>: Title shown in browser tab.
  • <body>: Main content of the page.

File Path

File path tells the browser where to find files like images, CSS, JavaScript, or other HTML files.

Types of File Paths

1. Absolute Path

<img src="https://example.com/images/pic.jpg" alt="Example Image">

Uses a full URL. File stored on a server.

2. Relative Path

<!-- Same folder -->
<img src="pic.jpg" alt="Same Folder">

<!-- Inside a subfolder -->
<img src="images/pic.jpg" alt="Subfolder Image">

<!-- One level up -->
<img src="../pic.jpg" alt="Parent Folder Image">

3. Root-relative Path

<img src="/assets/images/pic.jpg" alt="Root Folder Image">
